Introduction

Every technology is best learned from its official documentation.

https://git-scm.com/docs/git

Git Version

To check whether Git is installed on your system and to view its current version.

git -v

Git Init

Initializes a new Git repository in the current directory. It creates a hidden .git folder that starts tracking changes in the project. 

git init

Git Config

It is used to configure Git with your personal user information. 

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --list

Git Status

It shows the status of files in the working directory.

git status

Git Add

It adds files to the staging area to prepare them for a commit.

git add file.txt
git add .

Git Commit

It saves staged changes to the repository as a new snapshot.

git commit -m "your message"

Git Log

It is used to view the commit history of the project.

git log
git log --oneline
git log --graph --oneline

Git Ignore

It is used to specify files and folders that Git should ignore.

Example .gitignore file content.

node_modules/
.env
dist/
.DS_Store

Git Branch

It allows you to create separate lines of development within a project.

git branch
git branch feature
git switch feature
git checkout feature

Git Merge

It is used to combine changes from different branches.

git merge feature-login

Git Restore

It is used to discard changes made in files.

git restore file.txt
git restore .

Git Stash

It is used to temporarily save changes and clean the working directory.

git stash
git stash pop
git stash list
git stash drop
git stash apply stash@{0}
git stash clear

Navigating Between Commits

It is used to move between different commits in the project history to inspect old versions.

git checkout commit-hash

Git Reset

It is used to undo commits or move the project back to a previous state.

git reset commit-hash
git reset --soft commit-hash
git reset --hard commit-hash

Git Revert

It creates a new commit that undoes the changes from a previous commit.

git revert commit-hash
git reset --soft commit-hash
git reset --hard commit-hash

Git Diff

It is used to show line-by-line changes between file versions.

git diff
git diff --staged
git diff head
git diff commit1 commit2
git diff main feature-login

Git Rebase

It is used to move a branch's commits on top of another branch's latest state.

git switch feature-login
git rebase main

Git Remote

It is used to manage connections between a local repository and remote repositories.

git remote -v
git remote add origin https://github.com/user/repo.git
git remote set-url origin https://github.com/user/new-repo.git
git remote remove origin
git remote add origin https://github.com/user/repo.git

Git Fetch

It is used to download the latest changes from a remote repository without merging them.

git fetch
git fetch origin

Git Pull

It is used to fetch and merge changes from a remote repository into the current branch.

git pull
git pull origin main

Git Push

It is used to upload local commits to a remote repository.

git push
git push origin main
